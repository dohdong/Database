데이터와 정보는 다르다. 
데이터 : 측정한 값이나 사실   /   정보 : 측정한 값이나 사실+ 가공 => 의사결정가능하면 정보. 
			이런 정보를 file로 만들어서 grouping 한 것을 DB라고 한다.


데이터 베이스. 
    -> 구조화된 데이터의 집합.
	-> 1. 여러 사람과 실시간으로 공유하며 사용하기 위함.
	    2. 효율적인 데이터 관리, 검색
	    3. *일관성* 있는 방법으로 데이터 관리
	    4. 데이터 누락 및 *중복* 제거
				-> DB는 중복을 싫어함. 			 ex) 학과정보에도 전화번호가 있고,
				중복되면 data불일치 가능성이 있기 때문에	학생정보에도 있는 중복을 싫어함


파일 시스템과 DBMS 
파일 시스템은 학사 프로그램에 학사 데이터 따로 / 장학금 신청 프로그램에 장학금 데이터 따로
DBMS 는 학사 프로그램과 장학금 신청 프로그램이 하나의 => 재학생 관리 데이터베이스에서 관리함.


DBMS를 통한 데이터 관리. 
우리가 배울것은 DBMS 중에서 오라클을 이용해서 데이터베이스에 있는 정보를 꺼내올까. 활용할까


데이터 모델. 
계층형 데이터 모델 (tree 구조)  /  네트워크형 데이터 모델 (cycle 이 있음, 구현하기 어렵고 잘 안쓰긴함)
// 객체 지향형 모델 (교수님도 안써봄)

우리가 알아야 할 것은 *관계형 데이터 모델*임.(오라클, mysql, H~ 등등 전부 관계형 데이터 모델임)
DB는 행과 열이라고 부르지 않음.
열을 속성(attribute), 필드(field)라고 부르기도 함.
행을 tuple 이라고 부름. 

(24p)관계..? 는 사원과 부서의 관계 뭐 이런거
모든 것을 개체(entity)로 저장함. 이 개체들을 table로 저장한다. (= 자바의 object 라고 생각하면 됨. 객체같은 느낌)
( 전체를 통틀어서 사원entity   부서는 부서entity )       -> table로 저장하는 것이 관계형 데이터 모델의 특징이다.
사원 entity 에는 속성(attribute) 가 4개가 있음. 차수를 4차수 라고도 함. 튜플도 4개가 있음.
( Q. 사원entity 와 사원table 의 차이... 가 뭐지? 같은 거라면 흠.. 그냥 같은건가.. 뭐지? )

개체 하고 개체를 연결할때 속성을 가지고 연결함. (오 그렇지.. 24p 의 개체들을 봐도 부서코드 라는 속성으로 서로 연결가능) 

저장을 할때 (호봉 , 급여) , (사원번호, 이름, 호봉) 이렇게 두개로 나누어서 저장을 해야함. 호봉이라는 속성을 통해서 연결됨.    
(사원번호, 이름, 급여, 호봉) 전체x   , 또 중복이 되면 안됨.
